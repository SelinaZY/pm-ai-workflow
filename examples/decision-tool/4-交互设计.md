## P1 首次引导页

### 页面功能
首次打开 App 即为录入页（无独立欢迎页）。项目在后台自动创建，用户第一个操作直接产生价值——记录第一套房。与 P3 录入页共用同一页面，区别仅在于顶部标题和 AppBar 不同（P1 无「←」返回键）。

> **设计决策（D-01）**：采用方案 B，打开即录入。福格行为理论：高动机时刻最大化能力，消除中间页。

### ASCII 线框图

```
┌─────────────────────────┐
│  找房助手               │  ← 无返回键（首次进入）
│  ─────────────────────  │
│                         │
│  ┌───────────────────┐  │
│  │ 说说这套房怎么样， │  │  ← 文本框自动聚焦，键盘弹起
│  │ 或粘贴贝壳链接…   │  │
│  │                   │  │
│  │                   │  │
│  └───────────────────┘  │
│  🎤 语音输入            │
│                         │
│  ┌───────────────────┐  │
│  │   记录这套房      │  │  ← 主 CTA，有内容后激活
│  └───────────────────┘  │
└─────────────────────────┘
```

### 元素说明

| 元素 | 规则 |
|------|------|
| 顶部标题 | 「找房助手」，无「←」返回键 |
| 文本框 | 占屏幕主体，进入页面自动获取焦点、弹起键盘 |
| 语音按钮 | 调用系统语音输入，转文字后填入文本框 |
| 「记录这套房」按钮 | 文本框为空时禁用（灰色）；有内容后激活 |
| 项目自动创建 | 后台静默创建，默认名根据 IP 定位城市 + 当前年月；无定位权限则用「我的租房」 |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 初始 | 文本框空，按钮禁用，光标在文本框内，键盘弹起 |
| 输入中 | 按钮激活 |
| 粘贴链接 | 自动检测域名，文本框下方出现「正在解析链接…」loading 条 |
| 解析成功 | 字段预填至下方解析区，用户可修改后点「记录这套房」 |
| 解析失败 | 提示「链接解析失败，请手动描述这套房」，文本框保留内容 |
| 点击「记录这套房」 | 进入 AI 解析流程 → 解析完成后跳转 P2 房源库（新卡片出现） |

### 出口

| 操作 | 跳转目标 | 条件 |
|------|---------|------|
| 点击「记录这套房」 | P2 房源库 | 文本框有内容，AI 解析完成后自动跳转 |

## P2 房源库（主页面）

### 页面功能
展示本次找房项目中所有已记录的房源，支持发起对比、分享、进入标准面板。

### ASCII 线框图

```

│  ─────────────────────  │
│  候选中 (3)  已淘汰 (2) │  ← Tab 切换
│                         │
│  ┌─────────────────────┐│
│  │ 🏠 回迁房·长江路    ││  ← 房源卡片
│  │ 2900元·采光极好·车位好││
│  │ [候选中]            ││
│  └─────────────────────┘│
│  ┌─────────────────────┐│
│  │ 🏠 商品房·湖西街道  ││
│  │ 3200元·地铁近·装修好││
│  │ [候选中]  ☑         ││  ← 选中状态
│  └─────────────────────┘│
│  ┌─────────────────────┐│
│  │ 🏠 老公房·园区      ││
│  │ 2800元·安静·采光一般││
│  │ [候选中]  ☑         ││  ← 选中状态
│  └─────────────────────┘│
│                         │
│  ┌─────────────────────┐│
│  │    发起对比（2）    ││  ← 选中 2 张后浮现
│  └─────────────────────┘│
│                         │
│  [标准]        [+ 记录] │  ← 底部固定栏
└─────────────────────────┘
```

### 元素说明

| 元素 | 规则 |
|------|------|
| 项目名 | 默认「{城市}租房 {年月}」，点击可编辑（C2） |
| 分享按钮 📤 | 右上角，任意时刻可点击；点击后上传快照，生成链接 |
| Tab：候选中 / 已淘汰 | 默认显示「候选中」；已淘汰房源显示删除线样式 |
| 房源卡片 | 显示：名称/地址、AI 归纳 3 个关键标签（价格·最高评价维度·第二维度）、状态标签 |
| 多选模式 | 长按任意卡片进入多选模式；再次长按或点空白区域退出 |
| 「发起对比」浮层按钮 | 仅选中恰好 2 张卡片时出现；选 1 张或 3 张时不出现 |
| 「+ 记录」 | 底部固定，随时可点击 |
| 「标准」 | 底部固定，进入标准面板（P5） |
| 空状态 | 候选中无房源时：插画 + 「还没有记录房源，现在开始」按钮 |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 空状态（无房源） | 插画 + 引导文案 + 「开始记录第一套房」按钮 |
| 正常（有房源） | 卡片列表，底部操作栏 |
| 多选模式 | 卡片右上角出现复选框，已选卡片高亮边框 |
| 选中 2 张 | 底部浮现「发起对比（2）」按钮，覆盖原底部栏 |
| 分享上传中 | 分享按钮变 loading 态，禁止重复点击 |
| 分享成功 | 底部弹出 Toast「链接已生成，点击复制」+ 复制按钮 |
| 分享失败 | Toast「生成失败，请检查网络后重试」 |

### 出口

| 操作 | 跳转目标 | 条件 |
|------|---------|------|
| 点击「+ 记录」 | P3 录入页 | 无条件 |
| 点击「发起对比」 | P4 对比台 | 已选中恰好 2 张候选卡片 |
| 点击「标准」 | P5 标准面板 | 无条件 |
| 点击房源卡片（非多选模式） | P3 录入页（编辑模式） | 无条件 |

## P3 录入页

### 页面功能
现场快速录入一套房的印象，支持自由文字/语音/链接三种方式，AI 实时解析为结构化字段，用户确认后保存。

### ASCII 线框图

```
┌─────────────────────────┐
│ ←              记录房源  │
│  ─────────────────────  │
│                         │
│  ┌───────────────────┐  │
│  │ 这套房在长江路，  │  │  ← 自由文本区（主输入区）
│  │ 2900元，采光很好  │  │
│  │ 有小区，车位好租  │  │
│  └───────────────────┘  │
│  🎤  语音继续补充        │
│                         │
│  AI 解析结果：           │
│  ┌───────────────────┐  │
│  │ 价格      2900元  ✏️│  │
│  │ 采光      好      ✏️│  │
│  │ 车位      好      ✏️│  │
│  │ 地铁距离  —      ✏️│  │  ← 未提取到，用户可手填
│  │ 装修      —      ✏️│  │
│  │ + 添加其他维度       │  │
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │    保存这套房     │  │  ← 主 CTA
│  └───────────────────┘  │
└─────────────────────────┘
```

**链接解析中状态：**

```
┌─────────────────────────┐
│ ←              记录房源  │
│  ─────────────────────  │
│                         │
│  ┌───────────────────┐  │
│  │ https://ke.com/…  │  │
│  └───────────────────┘  │
│  ⏳ 正在解析链接…        │
│                         │
│  （解析完成后同上展示字段）│
└─────────────────────────┘
```

### 元素说明

| 元素 | 规则 |
|------|------|
| 自由文本区 | 默认显示 placeholder「说说这套房怎么样，或粘贴贝壳链接…」；进入页面自动聚焦 |
| 语音按钮 | 点击后追加语音转文字内容到文本区末尾，不覆盖已有内容 |
| AI 解析区 | 文本区有内容时，调用 LLM 解析，结果实时展示于下方；字段状态：好/一般/差，显示文字不用图标 |
| 字段编辑 ✏️ | 点击任意字段行可弹出简单编辑弹窗（文字输入 + 好/一般/差三选一） |
| 「+ 添加其他维度」 | 点击后弹出常用维度选择列表（噪音、小区环境、房东印象、步行体验等）+ 自定义输入 |
| 「保存这套房」按钮 | 文本区有内容即可点击（不强制等待 AI 解析完成） |
| 编辑模式（已有房源） | 顶部标题改为「编辑房源」，文本区预填原始记录，字段预填已有数据 |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 初始（文字录入） | 文本区空，AI 解析区不显示，保存按钮可点 |
| 输入中 | AI 解析区出现 loading，随后逐字段填入 |
| 链接粘贴 | 文本区显示链接，下方出现「正在解析链接…」条，字段区 loading |
| 解析成功 | 字段区展示预填结果 |
| 解析失败 | 提示「链接解析失败，请手动描述这套房」，文本区聚焦 |
| 保存成功 | 返回房源库（P2），新卡片出现在列表顶部 |
| 保存中（网络慢） | 按钮 loading 态，禁止重复点击 |

### 出口

| 操作 | 跳转目标 | 条件 |
|------|---------|------|
| 点击「保存这套房」 | P2 房源库 | 文本区有内容 |
| 点击左上角「←」 | P2 房源库 | 无条件，内容不保存（需二次确认：「确定放弃录入吗？」） |

## P4 对比台

### 页面功能
两套房源横向对比，按标准维度高亮差异，用户淘汰较差一套。

### ASCII 线框图

```
┌─────────────────────────┐
│ ←   对比                │
│  ─────────────────────  │
│         A        B      │
│    回迁房·长江路  商品房·湖西│
│    2900元        3200元  │
│  ─────────────────────  │
│  【一票否决】            │
│  小区环境  ✅好    ✅好  │
│                         │
│  【重要】               │
│  价格      ✅2900  ❌3200│
│  车位      ✅好    ❌需排队│
│  采光      ✅极好  ⚠️一般│
│  地铁      ❌1.2km ✅近  │
│                         │
│  【加分】               │
│  装修      ⚠️一般  ✅好  │
│  ─────────────────────  │
│  AI 摘要：A 价格和车位占优│
│  B 地铁和装修更好        │
│  ─────────────────────  │
│  [淘汰 A]      [淘汰 B] │
└─────────────────────────┘
```

**淘汰确认弹窗：**

```
┌───────────────────────┐
│ 淘汰「商品房·湖西街道」？│
│                       │
│ 原因（可选）：         │
│ ┌───────────────────┐ │
│ │ 价格太高，房东难合作│ │
│ └───────────────────┘ │
│                       │
│  [取消]   [确认淘汰]  │
└───────────────────────┘
```

**AI 标准建议气泡（对比完成后）：**

```
┌─────────────────────────────┐
│ 💡 你提到了「房东态度」      │
│    要加入你的决策标准吗？    │
│    [忽略]    [加入·重要级]  │
└─────────────────────────────┘
```

### 元素说明

| 元素 | 规则 |
|------|------|
| 维度分组 | 按标准权重分组：一票否决 → 重要 → 加分；无标准的维度归入「其他」 |
| 相对高亮 | ✅ = 相对更优；❌ = 相对更差；⚠️ = 有瑕疵但非最差；两者相同则不标注符号，字体置灰 |
| 维度不足提示 | 共同维度 < 3 时，页面顶部出现 banner：「这两套房的信息较少，建议补充后再对比」，不阻断操作 |
| AI 摘要（S1） | 对比台底部固定展示，字数控制在 2 行以内 |
| 「淘汰」按钮 | 两个并排，点击后弹出确认弹窗（含可选原因输入） |
| 淘汰原因（S2） | 文本框可选填，提交后供 M5 标准激活使用 |
| AI 标准建议（M5） | 淘汰确认后，若检测到新维度，底部弹出气泡提示；最多同时出现 1 条 |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 加载中 | 维度列表 skeleton loading |
| 正常 | 维度矩阵 + AI 摘要 + 淘汰按钮 |
| 维度不足 | 顶部 banner 提示，其余正常展示 |
| 淘汰弹窗 | 模态弹窗，背景变暗 |
| 淘汰成功 | 关闭弹窗 → AI 标准建议气泡（若有）→ 返回 P2 |

### 出口

| 操作 | 跳转目标 | 条件 |
|------|---------|------|
| 确认淘汰 | P2 房源库 | 点击「确认淘汰」 |
| 点击「←」 | P2 房源库 | 无条件，不触发淘汰 |
| 点击 AI 气泡「加入标准」 | 停留当前页，标准写入 P5 | 气泡显示时 |

## P5 标准面板

### 页面功能
管理本次找房的决策标准，支持查看、调整权重级别、处理 AI 建议。

### ASCII 线框图

```
┌─────────────────────────┐
│ ←   我的找房标准        │
│  ─────────────────────  │
│                         │
│  💡 AI 建议             │
│  ┌─────────────────────┐│
│  │ 你多次提到「步行体验」││
│  │ 要加入标准吗？       ││
│  │ [忽略]  [加入·重要级]││
│  └─────────────────────┘│
│                         │
│  【一票否决】            │
│  · 小区环境    [切换 ▾] │
│                         │
│  【重要】               │
│  · 价格        [切换 ▾] │
│  · 车位        [切换 ▾] │
│  · 采光        [切换 ▾] │
│  · 地铁距离    [切换 ▾] │
│                         │
│  【加分】               │
│  · 装修        [切换 ▾] │
│                         │
│  + 手动添加标准          │
└─────────────────────────┘
```

**切换权重弹窗：**

```
┌───────────────────────┐
│  调整「采光」的重要程度  │
│                       │
│  ○ 一票否决           │
│  ● 重要               │
│  ○ 加分               │
│                       │
│  [取消]    [确认]     │
└───────────────────────┘
```

### 元素说明

| 元素 | 规则 |
|------|------|
| AI 建议区 | 页面顶部，有待处理的 AI 建议时显示；无建议时隐藏该区域 |
| 标准列表 | 按一票否决 / 重要 / 加分三组展示；组内按激活时间排序 |
| 「切换 ▾」 | 点击弹出三选一弹窗，选择后立即保存并重新分组 |
| S3 重算 | 切换权重后，若当前有上次对比的记录，对比台维度顺序自动更新（后台静默，不跳转页面） |
| 「+ 手动添加标准」 | 输入维度名称，默认加入「重要」组 |
| 空状态 | 无任何标准时：「AI 会在你记录房源时自动归纳标准，也可以手动添加」 |

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 有 AI 建议 | 顶部显示建议卡片 |
| 无 AI 建议 | 建议区隐藏，标准列表顶格显示 |
| 空状态（无标准） | 插图 + 说明文案 + 「手动添加」按钮 |
| 切换权重弹窗 | 模态弹窗 |
| 权重切换成功 | 弹窗关闭，标准重新分组动画 |

### 出口

| 操作 | 跳转目标 | 条件 |
|------|---------|------|
| 点击「←」 | P2 房源库 | 无条件 |

### 元素说明

| 元素 | 规则 |
|------|------|
| 快照时间 | 显示「快照时间：XX月XX日 XX:XX」，说明数据非实时 |
| 候选中 | 展示所有候选中房源的对比矩阵（横向滚动）；**最后一次对比的两列加粗边框高亮**，其余列降权（标题灰色）引导视线聚焦 |
| 已淘汰 | 默认折叠，点击「展开」后显示被淘汰房源卡片列表（不展示对比矩阵） |
| 相对高亮 | 同 P4 对比台规则（✅/❌/⚠️） |
| 所有元素 | 只读，无任何编辑入口 |
| C1 拉新入口 | 页面底部固定，文案「用同款工具找房 →」，点击跳转 H5 首页 |

> **设计决策（D-02）**：候选房源 > 2 套时横向滚动展示所有列（方案 A），但高亮最后一次对比的两列。福格行为理论：接收方动机较低，需要完整信息才能完成「帮忙判断」的任务，仅展示 2 套会造成信息缺失，反而增加沟通成本。

### 状态矩阵

| 状态 | 表现 |
|------|------|
| 正常 | 完整展示快照内容 |
| 候选房源仅 1 套 | 仅展示单套房源信息，不展示对比矩阵，改为卡片详情视图 |
| 链接过期（服务端删除） | 显示「该分享链接已过期」 |
| 加载失败 | 显示「加载失败，请检查网络」 |

### 出口

| 操作 | 跳转目标 | 条件 |
|------|---------|------|
| 点击「用同款工具找房 →」 | H5 首页（P1 或 P2） | C1 功能启用时 |
