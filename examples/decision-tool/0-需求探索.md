## 关键决策

| 维度 | 决策 |
|------|------|
| MVP 范围 | 只做租房 |
| 平台 | 移动端优先（H5） |
| 协作 | 只读分享链接（无多人编辑） |
| AI | 辅助解析 + 标准归纳 |
| 核心交互模式 | 锦标赛式对比（A vs B 淘汰制） |
| 录入方式 | 语音/文字随手记，AI 结构化 |
| 标准管理 | 从行为浮现，可随时调权重 |
| 数据存储 | 本地优先（IndexedDB），无需登录 |

---

## 核心洞察

1. **锦标赛式决策**：用户不是给所有房源打分后排名，而是「A vs B 淘汰一个，胜者 vs C」。核心交互单元是两套房的横向对比，不是列表。

2. **标准动态演化**：
   - 隐性标准：看到反例才激活（如「小区环境」直到第一套房才意识到）
   - 权重漂移：过程中主动降级（如「装修质量」重要性下降）
   - 因此标准不能预先设定，必须从行为中浮现

3. **现场零门槛**：用户在看房现场不愿意上手任何需要学习的工具。目标是拿出手机 → 一步操作 → 开始记录。

## 信息架构

产品由三个区域构成：

### ① 房源库（主页面）
- 卡片列表展示所有已记录房源
- 每张卡片：房源名/地址、AI 归纳的 3 个关键标签（如「2900元·采光极好·车位好租」）、状态（候选中 / 已淘汰）
- 右上角「+ 记录新房源」入口

### ② 对比台（核心交互）
- 任意选两套房源发起对比
- 横向展示关键维度差异，差异项自动高亮，平手项置灰
- 底部「淘汰哪套？」操作，点击后需输入一句话原因（可选）
- 淘汰原因供 AI 归纳标准使用
- AI 在对比后提示新发现的标准维度

### ③ 标准面板（辅助工具）
- 展示已激活标准，分三级：一票否决 / 重要 / 加分
- 可拖拽调整权重，调整后对比台自动刷新评分
- AI 建议区：「你在 3 套房里都提到了步行体验，要加入标准吗？」

## 核心流程

```
记录房源（语音/文字/链接）
    ↓
AI 解析 → 结构化字段（用户可修改）
    ↓
发起 A vs B 对比 → 淘汰一套（记录原因）
    ↓
胜者 vs 下一套 → 继续淘汰
    ↓
最终 1-2 套候选 → 生成只读分享链接
```

---

## 核心交互细节

### 录入房源（现场场景）

目标：**现场 60 秒以内完成一套房的录入**。

- 顶部大文本框占屏幕主体，placeholder：「说说这套房怎么样，或者粘贴贝壳链接…」
- 支持语音输入（调用系统麦克风）
- AI 实时解析，底部逐渐出现结构化字段预览：价格、地址、采光、车位、装修、噪音等
- 用户可点击字段直接修改 AI 解析结果
- 「保存」按钮完成录入

### 对比台交互

```
维度          房源 A（回迁房）    房源 B（商品房）
价格          ✅ 2900            ❌ 3200
车位          ✅ 好租            ❌ 需排队
采光          ✅ 全屋极好        ⚠️ 一卧差
地铁距离      ❌ 1.2km路难走     ✅ 近
装修          ⚠️ 一般            ✅ 更好
```

- ✅ 优势 / ❌ 劣势 / ⚠️ 有瑕疵
- 底部「淘汰 A」/「淘汰 B」按钮
- AI 对比摘要：「A 的核心优势是价格和车位，B 的核心优势是地铁和装修。」

### 标准动态激活

触发时机：
1. **对比后**：检测到新维度 → 气泡提示「你刚才提到了房东态度，要加入标准吗？」→ 一键加入
2. **反复出现**：某个词在 3 条以上记录里出现 → AI 主动建议加入

## AI 能力边界（MVP）

| 功能 | 实现方式 | 备注 |
|------|---------|------|
| 解析自由文本 | LLM 提取结构化字段 | 价格/地址/各维度评价 |
| 解析贝壳链接 | 爬取页面内容 → LLM 归纳 | 需处理反爬风险 |
| 归纳隐性标准 | 高频词统计 + LLM 判断 | 出现 3 次以上触发 |
| 对比摘要 | LLM 生成一段差异描述 | 对比台底部展示 |

**AI 不做**：替用户做决策。只呈现差异、归纳标准、提供摘要，最终判断由用户完成。

## 数据结构

```
决策项目（一次找房 = 一个项目）
  ├── 标准列表
  │     ├── 维度名称
  │     ├── 权重（一票否决 / 重要 / 加分）
  │     └── 激活方式（用户手动 / AI 建议）
  └── 房源列表
        ├── 原始记录（用户原文）
        ├── 结构化字段（AI 解析结果）
        └── 淘汰状态 + 淘汰原因
```

存储：IndexedDB（本地优先，无需登录）。

---

## 分享页

只读静态页面，展示：
- 项目名称（如「苏州租房 2026-04」）
- 候选房源对比矩阵
- 已激活标准列表
- 底部「用同款工具找房」入口（拉新）

分享时，将当前项目数据序列化后上传至服务端生成只读快照页（无需用户账号，后端仅存储快照数据）。接收方打开链接看到的是该时刻的静态内容，快照不可编辑、不与本地数据同步。
