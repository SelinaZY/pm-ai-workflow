# PRD：风格画布（StyleCanvas）v0.1

> 上游：`1-需求调研.md`（GO）、`0-需求探索.md`

## 一、背景

PM/设计师在早期方案沟通中，需要快速把界面想法变成高保真稿并讲清多界面流程。现状是"从零画太慢 / 线框不够真 / 单页工具摆不到一起 / 风格易漂移"。

## 二、目标与成功指标

| 目标 | 指标 |
|------|------|
| 大幅降低出高保真稿的时间 | 一句话生成首个可用 frame ≤ 30s |
| 风格统一可控 | 同画布内 frame 风格一致率（主观评分）≥ 90% |
| AI 改写可控不翻车 | AI 改写采纳率 ≥ 60%，且"误覆盖"投诉为 0（强制 diff） |
| 真正用于沟通 | 周活用户中 ≥ 40% 使用过"连线/注记"做流程讲解 |

## 三、用户与场景

- **主用户**：B 端产品经理、交互/产品设计师。
- **核心场景**：评审前 30 分钟，把 3–5 个界面 + 流程在一块画布上摆出来。

## 四、功能范围（MVP）

### F1 一句话生成界面 frame
- 顶部 prompt 输入框 + 风格选择器 + 「生成界面」。
- 输出：画布上新增一个高保真 frame（含真实排版的 mini-UI），带"AI 生成 · {风格名}"标签。
- 生成中显示占位 frame + 流式状态。

### F2 设计风格库（差异化核心）
- 内置风格：文档/画布风、企业后台风、移动端 App、营销落地页（每个风格 = 一组色/字/间距/圆角/组件规范）。
- 生成前选定风格；同一画布默认沿用同一风格，保证统一。
- 复用 prototype 技能的"按设计参考生成"机制（`design-references/`）。

### F3 选中 frame 的局部 AI 操作
- 选中 frame → 浮动工具栏：「✦ AI 改写」「换风格」「复制」「连线」。
- **AI 改写**：弹出指令（如"改成深色 + 加第三方登录"）→ 生成**改写预览**。
- **换风格**：一键把该 frame 切换到风格库里的另一种设计语言。

### F4 AI 产出的确定性边界（强制）
- AI 改写**不直接覆盖**原 frame，以 **diff 卡片**呈现（原 → 新，逐条）。
- 三个动作：**采纳 / 仅看差异 / 重新生成**；采纳前原 frame 不变；采纳后**可回退**。

### F5 画布编排
- 无限画布：拖拽 frame、缩放、平移、minimap。
- 连线（流程箭头）把 frame 串成跳转流程。
- 文本注记 / 评论 @人（用于评审）。

### F6 右侧 AI 助手面板
- 对话式，可 @ 某个 frame 下指令；可承载多轮复杂修改。

## 五、非功能需求

- 生成首屏 ≤ 30s；画布 50 个 frame 内操作不卡顿。
- 风格库可扩展（新增一种风格 = 新增一份设计参考，不改主流程）。

## 六、不做（本期边界）

- 导出 Figma / 生产代码；真实多人实时协同；用户上传自定义风格。

## 七、关键交互规则

- 同画布新建 frame 默认继承当前风格；可在生成时临时改风格。
- 任何 AI 写操作（生成除外的"改"）一律走 diff + 接受/拒绝。
- 删除/改写均可回退（操作历史）。

## 八、用户场景视角验收要点（节选）

1. **作为 PM**，我输入一句话并选"文档/画布风"，能在 30s 内得到一个排版真实的高保真 frame。
2. **当**我要把登录页改成深色并加第三方登录，AI 给我 diff 而**不是**直接改掉，我看完才采纳。
3. **当**我把登录、看板两个 frame 连线，评审时能直接对着画布讲清跳转。

## 九、依赖

- AI 生成服务（文本→UI）；风格库 / 设计参考体系（已有 prototype 技能基础）。

## 十、里程碑（建议）

- M1：F1+F2 主链路（生成 + 风格）；M2：F3+F4（局部 AI 改写 + diff）；M3：F5+F6（画布编排 + AI 面板）。

回复「继续」进入阶段三：需求评审。
