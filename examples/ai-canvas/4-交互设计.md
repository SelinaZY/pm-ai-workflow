> 上游：`2-PRD.md`、`3-需求评审.md`（R1–R3 已在下方对应处落定）
> 设计参考：`.claude/skills/prototype/design-references/document-canvas.md`（画布型布局）

## 一、页面/区域清单

| 编号 | 区域 | 说明 |
|------|------|------|
| A 顶栏 | prompt 输入 + 风格选择器 + 生成按钮 | F1 入口 |
| B 左栏 | 设计风格库 + 图层 | F2 |
| C 画布 | 无限画布：frame / 连线 / 注记 / 工具坞 / 缩放 / minimap | F1/F5 |
| D 右栏 | AI 助手（对话 + diff）/ 属性 / 历史 | F3/F4/F6 |
| E 浮层 | 选中态浮动工具栏、斜杠/AI 生成 popover、diff 卡片 | F3/F4 |

## 二、整体布局（画布型，三栏 + 浮层）

```
┌───────────────────────────── A 顶栏 ─────────────────────────────┐
│ ◳ 风格画布   [✦ 描述一句话…………]  [●文档/画布风▾] [✦生成]   分享 (头像) │
├──────────┬────────────────────────────────────────┬──────────────┤
│ B 左栏    │              C 画布（点阵背景）           │ D 右栏        │
│ 风格库    │   ┌登录页┐····连线····┌─数据看板─┐         │ ✦AI助手|属性  │
│ ·文档画布 │   │mini-UI│           │ mini-UI  │         │ ┌ diff 卡片┐ │
│ ·企业后台 │   └───────┘           └──────────┘         │ │原→新      │ │
│ ·移动端   │   ┌生成中…┐  💬评审注记                    │ │采纳/差异/ │ │
│ ·营销页   │   └────────┘                              │ │重新生成   │ │
│ 图层      │  [⌖✋T▢↘✦]坞              [minimap][−100%+]│ └──────────┘ │

```

## 三、核心流程

### 流程 1：一句话生成（F1）
```
顶栏输入描述 → 选风格 → 点「生成界面」
   → 画布出现"生成中"占位 frame（流式状态 + 取消[G1]）
   → 完成：占位替换为高保真 frame，带「AI 生成·{风格}」标签，自动选中
```

### 流程 2：选中 frame 局部 AI 改写（F3+F4，确定性边界）
```
点选 frame → 浮出工具栏[✦AI改写|换风格|复制|连线]
   → 点「✦AI改写」→ 输入指令（或在右栏 @该 frame 下指令）
   → 右栏生成 diff 卡片（区块级[R2]：背景/组件/新增项 逐条 原→新）
   → 三选一：
       · 采纳   → 应用到 frame，入撤销栈（可回退[Y3]）
       · 仅看差异 → 高亮对比，不应用
       · 重新生成 → 换一版 diff
   ※ 采纳前原 frame 始终不变
```

### 流程 3：画布编排（F5）
```
拖拽 frame 排布 → 从 frame 边缘拖出连线[G3] → 串成跳转流程
                → T 文本注记 / 评论 @人（评审用）
```

## 四、跳转/触发规则

| 触发 | 结果 |
|------|------|
| 顶栏「生成」 | 按当前风格在画布新增 frame（继承画布默认风格[Y2]） |
| 单击 frame | 选中 + 浮出工具栏；右栏切到该 frame 上下文 |
| 工具栏「换风格」 | 该 frame 切换为所选风格（frame 自身风格优先于画布默认[Y2]） |
| 任意"AI 改写/换风格" | 一律走 diff 预览，不直接覆盖[R2] |
| 撤销（⌘Z） | 单步回退最近一次应用[Y3] |

## 五、状态矩阵

| 对象 | 状态 | 表现 |
|------|------|------|
| frame | 默认 / 选中 / 生成中 / AI改写待确认 | 选中=紫色外框；生成中=虚线+spinner+流式；待确认=右栏 diff 未采纳 |
| 生成 | 进行中 / 成功 / 超时 | 超时→骨架 + 重试[Y1]，不卡死 |
| diff | 待决策 / 已采纳 / 已弃 | 采纳后入历史、可回退；弃则原样 |
| 风格项 | 选中 / 未选 | 选中=紫底高亮 + 左侧色条 |

## 六、设计语言要点（落定 R1/R3）

- 遵循 `document-canvas` 画布型：点阵画布、浮动工具条、minimap、缩放控制。
- **AI 相关元素统一用紫色**（`--ai:#8B5CF6`）+ 左色条/标签，与品牌蓝区分——让"哪些是 AI 产出"一眼可辨。
- "一种风格"的 schema（R3）：色板 / 字体 / 间距 / 圆角 / 阴影 / 基础组件清单，与 `design-references/<风格>.md` 一一对应。

> 对应高保真原型：`prototype/index.html`
