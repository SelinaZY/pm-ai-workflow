# 页面现状采集

## 使用方式

**自动模式 — 路由采集**（推荐，一次采集多个页面）：

```
采集页面 auto <产品名> all                    ← 采集该产品所有已配置页面
采集页面 auto <产品名> <页面1>,<页面2>         ← 采集指定页面
采集页面 auto <产品名> <页面名>                ← 采集单个页面
```

脚本从产品入口 URL 出发，按预配置的导航路径自动点击跳转到各页面并采集 DOM。用户只需提供产品名，无需提供每个页面的 URL。

导航路由配置文件：`.claude/skills/capture-page/capture-routes.json`

**自动模式 — URL 直连**（适用于未配置路由的页面）：

```
采集页面 auto-url <URL> <页面名>
```

**手动模式**（用户自己贴 DOM）：

```
采集页面 <产品名> <页面名>
```

然后在输入框中粘贴从浏览器 F12 → Elements → 右键 Copy element 得到的 HTML DOM 片段。也可以同时附上截图辅助校对布局。

## 交互规则（严格遵守）

### Step 1：判断模式并解析输入

从 `$ARGUMENTS` 的第一个词判断模式：

**若第一个词是 `auto`**：进入路由自动模式
- 解析产品名（第二个词）、页面范围（第三个词，可以是 `all`、逗号分隔的页面名、或单个页面名）
- 若产品名缺失，提示用户补充
- 若页面名缺失，默认采集该产品所有已配置页面

**若第一个词是 `auto-url`**：进入 URL 直连模式
- 解析 URL（第二个词）、页面名（第三个词）
- 若任一缺失，提示用户补充

**否则**：进入手动模式
- 解析产品名（第一个词）、页面名（第二个词）
- DOM 内容为其余部分
- 若产品名或页面名缺失，提示用户补充
- 若 DOM 内容为空或不含 HTML 标签，提示用户：
  ```
  未检测到 DOM 内容。请在浏览器中：
  1. F12 打开 DevTools → Elements 面板
  2. 选中页面主内容区的根节点
  3. 右键 → Copy → Copy element
  4. 将复制的 HTML 粘贴在产品名和页面名后面
  ```

### Step 2（仅自动模式）：运行 Playwright 提取 DOM

**路由模式**执行：

```bash
node .claude/skills/capture-page/scripts/capture-dom.mjs "<产品名>" "<页面名|页面1,页面2|all>"
```

**URL 直连模式**执行：

```bash
node .claude/skills/capture-page/scripts/capture-dom.mjs "<URL>" "<页面名>"
```

脚本行为（路由模式）：
1. 读取 `.claude/skills/capture-page/capture-routes.json` 获取产品入口 URL 和页面导航路径
2. 使用 MSEdge 打开产品入口页面
3. 加载已保存的登录态，过期则等待用户手动登录
4. 对每个目标页面：从入口出发，按配置的导航步骤（点击按钮等）自动跳转到目标页面
5. 到达目标页面后提取主内容区 DOM，保存到 `products/<产品名>/pages/dom/<页面名>.html`
6. 回到入口，继续采集下一个页面

**超时设置**：导航超时 120 秒，适配慢网络。

**登录说明**：目标产品（以手机号 + 验证码登录的 SaaS 为例），Token 有效期 24 小时。脚本会保存登录态到 `.claude/.auth/auth-state.json`，24 小时内免登录。过期后脚本会自动检测并弹出浏览器等待用户手动登录（90 秒轮询），登录完成后自动继续采集。若 90 秒内未完成登录，可通过 `LOGIN_WAIT=180` 环境变量延长等待时间。

**脚本执行后**：
- 读取 `products/<产品名>/pages/dom/<页面名>.html` 作为 DOM 内容
- 继续执行后续步骤

**若脚本失败**：
- 输出错误信息，建议用户改用手动模式
- 常见原因：网络超时、登录态过期、页面 URL 变化

### Step 3：检查已有文档

检查 `products/<产品名>/pages/<页面名>.md` 是否存在：

- **已有文档**：读取全文，提取「业务规则」章节及任何人工补充的内容，后续生成时保留这些内容
- **无文档**：标记为新建模式

### Step 4：解析 DOM 生成页面文档

从 HTML DOM 中提取以下信息，**不要编造 DOM 中不存在的内容**：

1. **页面结构**：从 DOM 层级推导出页面区域树形图
2. **各区域元素**：
   - 表单字段：`<input>` / `<select>` / `<textarea>` 等，提取 placeholder、label、type、disabled 状态
   - 表格列：`<th>` / `<td>` 结构，提取列名、数据类型
   - 按钮/操作：`<button>` / `<a>` 等，提取文案、样式（主操作/次操作/图标按钮）、disabled 状态
   - 状态标签：带颜色/样式的 Tag、Badge 等，提取文本和颜色
   - 导航元素：Tab、面包屑、侧边栏等
3. **枚举值**：从 `<select>` 选项、状态标签变体、Tab 文案等提取
4. **可推断的业务规则**：disabled 条件、必填标识（`*` / `required`）、字段校验提示等

**输出格式**严格遵循现有 `products/*/pages/` 文件的格式：

```markdown
# {产品名} · 页面现状：{页面中文名}

> 来源：{DOM 结构采集 / Playwright 自动采集}
> 状态：线上态
> 采集 URL：{URL}（仅自动模式）
> 最后更新：{YYYY-MM-DD}

---

## 一、页面结构

（树形图，使用 ├── └── 格式）

## 二、{区域名称}

（表格形式描述各元素）

## ...（按区域逐个描述）

## N、业务规则

（从 DOM 推断的规则 + 保留已有文档中的人工补充规则）
```

**格式参照**：在生成前读取 `products/<产品名>/pages/` 下的任意一个已有文件，对齐章节结构和表格风格。

### Step 5：展示变更 diff 并等待用户确认

**新建模式**：展示完整文档概要，确认后直接写入 `products/<产品名>/pages/<页面名>.md`。

**更新模式**：对比生成的新文档与现有文档，输出变更预览：

```
📝 页面知识库更新预览（线上态）

页面：{页面名}.md
变更项：
  + 新增：{描述}
  ~ 修改：{描述}
  - 删除：{描述}
  = 保留（人工补充内容）：{描述}

回复「确认」写入，「跳过」不更新，或指出需要调整的地方。
```

用户确认后才写入文件。写入后重新读取文件，确认内容完整无误。

若现有文档状态为「设计态」，Playwright 采集更新后自动改为「线上态」。

### Step 6：输出摘要

```
✅ 页面现状文档已生成
📄 products/{产品名}/pages/{页面名}.md
📝 模式：{新建 / 更新}
🔧 采集方式：{自动（Playwright） / 手动（DOM 粘贴）}

提取到的结构：
- 区域数：[N] 个
- 表单字段：[N] 个
- 表格列：[N] 列
- 按钮/操作：[N] 个
- 枚举值：[N] 组

⚠️ 以下信息需人工补充（DOM 无法获取）：
- [ ] 枚举值的业务含义（如状态标签的中文解释）
- [ ] 跳转目标（按钮点击后去哪个页面）
- [ ] 业务规则（如可导出条件、权限控制等）
- [ ] 字段间的联动关系

可贴截图辅助校对布局准确性。
```

## 注意事项

- **只从 DOM 中提取可见信息**，不推测页面功能或编造不存在的元素
- DOM 中的 CSS class 名可作为辅助判断（如 `btn-primary` 表示主操作），但不写入文档
- 若 DOM 片段过小（只是页面的局部），在摘要中提示用户可能需要补充其他区域
- 若用户同时提供了截图，结合截图验证 DOM 解析出的布局是否正确
- 自动模式的 DOM 可能很大，解析时聚焦页面主体内容区，忽略全局导航和页脚等通用区域
