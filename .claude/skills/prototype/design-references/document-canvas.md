# 设计参考：桌面文档 / 画布类工具

> 用途：当一个**全新产品（无线上 DOM 可采）**属于「文档编辑器 / 智能画布 / 知识协同」类型时，prototype 技能以本文件为设计语言生成 HTML 原型，替代默认的 Ant Design 后台风格。
> 对标产品：Notion、Linear、飞书文档、语雀、Figma、tldraw、Craft。

---

## 0. 一句话设计哲学

**内容是主角，工具退场。** 文档/画布工具的好坏不取决于控件多漂亮，而取决于排版是否舒适、界面是否克制、操作是否"在手边出现、用完即隐"。能用留白和排版解决的，就不要用边框和按钮。

---

## 1. 设计原则（生成原型时逐条对照）

1. **排版优先（Typography-first）**：正文是产品的灵魂，优先保证阅读舒适度，而非信息密度。
2. **克制配色**：大面积中性灰阶 + 大留白，品牌色只用于「选中 / 链接 / AI 高亮」等少数点缀。避免后台管理系统的高饱和色块。
3. **低 chrome（Low-chrome）**：工具栏隐藏式——默认不显示，选中内容或 hover 时才浮出。用斜杠命令、内联操作替代满屏按钮。
4. **内容区限宽**：文档正文最大宽度约 680–760px 居中，避免长行难读；画布则相反，铺满视口。
5. **适中偏松的密度**：编辑器行距、块间距比后台表格松；不是 Ant Design 的紧凑表格密度。
6. **微交互克制**：hover 浮起、淡入淡出、骨架屏、乐观更新；不用大幅颜色突变或弹跳动画。

## 2. 布局骨架

### 文档型（Document）

```
┌──────────┬─────────────────────────────────────┬────────────┐
│ 侧边导航  │            主编辑区                   │  右侧面板   │
│ 220–260px│   （内容区限宽 ~720px 居中）          │  240–320px │
│          │                                      │（可收起）  │
│ · 工作区  │   标题（大号，32–40px）               │ · 大纲/目录 │
│ · 文档树  │   ────────────                       │ · 属性/评论 │
│ · 收藏    │   正文块（段落/标题/列表/表格/代码）   │ · AI 助手  │
│          │   ……                                 │            │
│ 底部:账户 │                                      │            │
└──────────┴─────────────────────────────────────┴────────────┘
```

- 三栏：侧边导航 + 主编辑区 + 右侧面板（大纲 / 属性 / AI），右侧面板可收起。
- 顶部极简：面包屑 + 分享/更多，**不放**满排功能按钮。
- 主编辑区背景纯净（白或极浅暖灰），内容区限宽居中。

### 画布型（Canvas）

```
┌─────────────────────────────────────────────────────────┐
│  ┌─────────┐                              ┌───────────┐  │
│  │左侧图层  │        无限画布               │ 右侧属性   │  │
│  │/资源面板 │   （节点 + 连线 + 自由排布）   │  面板      │  │
│  └─────────┘                              └───────────┘  │
│                  ┌──────────────────┐                     │
│                  │ 底部浮动工具条     │      [minimap]      │
│                  │ 选择/手型/文本/形状│                     │
│                  └──────────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

- 无限画布铺满视口；工具以**浮动工具条 + 上下文菜单**形式出现，不占固定栏。
- 必备：缩放层级控制、对齐吸附、多选框、右下角 minimap。

## 3. 排版系统（最重要）

| 用途 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 文档大标题 H1 | 32–40px | 700 | 1.2 |
| 章节标题 H2 | 24–28px | 600 | 1.3 |
| 小标题 H3 | 18–20px | 600 | 1.4 |
| **正文** | **15–16px** | 400 | **1.6–1.7** |
| 次要/注释 | 13–14px | 400 | 1.5 |

- 中文：`"Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`
- 英文/数字/代码：`"Inter", "SF Mono", monospace`
- 正文内容区限宽 **~720px**，段落间距 ≥ 0.6em，标题上间距 > 下间距（制造章节呼吸感）。

## 4. 色彩系统

```
背景        #FFFFFF / #FBFBFA（极浅暖灰，降疲劳）
正文        #1F2328
次要文字    #6B6F76
最弱/占位    #9CA3AF
分割线      #EBECED（极细，1px，克制）
品牌强调色   单一一个（如 #2563EB 蓝 或产品自有色），仅用于链接/选中/主按钮
AI 高亮     一个区别于品牌色的色（如 #8B5CF6 紫渐变），专门标识"AI 生成/AI 介入"区域
语义        成功 #16A34A · 警告 #D97706 · 错误 #DC2626（低饱和）
```

> 关键：**AI 相关元素用独立的高亮色/渐变**，让用户一眼分清"这是 AI 产出的"，这是文档/画布类 AI 工具的标志性视觉语言。

---

## 5. 核心组件

- **编辑块（Block）**：段落、标题、列表、待办、引用、代码块、表格、分割线、图片、Callout 提示框。块左侧 hover 出现「⋮⋮ 拖拽手柄 + ＋ 新增」。
- **斜杠命令菜单（/）**：空块输入 `/` 弹出命令面板（插入标题/列表/表格/代码/AI…），带搜索和图标。
- **浮动工具栏（Floating Toolbar）**：选中文本时浮出——加粗/斜体/链接/颜色/**「✦ 用 AI」**。默认不可见。
- **侧边面板**：大纲（可点击跳转）、属性、评论、AI 助手，标签页切换，可整体收起。
- **画布节点 / 连线**：圆角卡片节点、贝塞尔连线、选中态外框 + 控制点、hover 高亮可连接点。
- **空状态**：友好引导 + 一个主行动（如「输入一句话，让 AI 起草」），不要空白表格。

## 6. ✦ AI 介入的交互范式（核心，也是面试高频考点）

文档/画布类 AI 工具的设计精髓在于"AI 怎么嵌进内容流"。四种范式按场景选用：

| 范式 | 触发 | 形态 | 适用 |
|------|------|------|------|
| **内联 AI** | 选中文本 → 浮动工具栏「✦ 用 AI」 | 原地弹出指令框（改写/润色/扩写/翻译） | 对已有内容做局部加工 |
| **块级 AI** | 空块输入 `/AI` 或回车呼出 | 块内生成，流式打字效果 | 从零起草一段/一节 |
| **侧边 AI 面板** | 右侧 AI 标签 | 对话式，可引用文档/画布内容 | 多轮、需要上下文的复杂任务 |
| **画布级 AI** | 选中节点/区域 → 上下文菜单 | 基于选中节点生成/重排/连线 | 画布的结构化生成 |

**AI 产出的确定性边界（必须设计，面试会追问）：**

1. **生成态可视**：AI 正在生成时用独立高亮色 + 流式打字 + 「停止」按钮。
2. **接受 / 拒绝 / 重试**：AI 产出**不直接落定**，而是以"待确认"态呈现，提供「采纳 / 丢弃 / 重新生成」三个动作。
3. **Diff 对比**：对"改写已有内容"类操作，并排或行内展示「原文 → AI 改写」的差异，让用户看清改了什么再决定。
4. **可追溯**：标注哪些内容由 AI 生成/修改（如左侧色条），支持一键回退。

> 设计主张可背诵：**"AI 负责把草稿写到 80 分，但落定权永远在用户手里——所以每一处 AI 产出都必须可预览、可对比、可拒绝、可回退。"**

## 7. 实现提示（生成 HTML 原型时）

- 字体用上面字栈；英文/代码用 Inter。
- 用 CSS 变量定义上面的色板与间距，集中在 `:root`。
- 正文容器 `max-width: 720px; margin: 0 auto;`。
- 浮动工具栏 / 斜杠菜单用 `position: absolute` + 阴影 `0 4px 16px rgba(0,0,0,.08)`，默认 `display:none`，交互时显示。
- AI 区域用独立高亮色 + 左侧 3px 色条标识；生成态加打字光标动画。
- 画布用一个大 `div` + `transform: scale()` 模拟缩放；节点用绝对定位卡片。
