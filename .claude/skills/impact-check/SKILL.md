# 变更影响检查

## 触发条件

- 用户说「影响检查」「impact check」「检查影响」
- 流程规则或需求产物改动后，主动建议用户执行

## 输入

从用户描述或当前会话上下文中识别：

1. **变更文件列表**：哪些文件刚被修改（可通过 `git diff --name-only` 获取）
2. **变更类型**：自动分类为以下两种之一
   - **流程变更**：修改了 skill 文件（`.claude/commands/*.md`、`.claude/skills/*/SKILL.md`）或 `CLAUDE.md` 中的流程规则
   - **产物变更**：修改了 PM 产物（`workspace/` 下的调研报告、PRD、评审报告、IxD）

## 执行逻辑

### 流程变更 → 查 Skill 依赖矩阵

1. 读取 `.claude/commands/pm-workflow.md` 中的「Skill 文件依赖关系」章节
2. 根据被改文件，找出矩阵中的所有关联文件
3. 对每个关联文件，用 Grep 搜索与变更内容相关的关键词（如阶段衔接语「继续」「自动进入」、流程描述等）
4. 输出需检查的位置

### 产物变更 → 查上下游产物

依赖关系图：

```
brainstorming 设计文档
    ↓
调研报告（背景、JTBD、成功标准、竞品数据）
    ↓
PRD（功能范围、字段定义、流程步骤、验收标准）
    ↓                    ↓
评审报告（引用 PRD 章节）   IxD（功能范围、字段、流程）
                              ↓
                         pages/ 知识库（设计态字段、枚举值）
```

1. 根据被改文件在依赖图中的位置，确定需要检查的下游文件
2. 提取变更的关键内容（字段名、枚举值、流程步骤名、规则描述等）
3. 在下游文件中 Grep 搜索这些关键词
4. 输出匹配位置

## 输出格式

```
🔍 变更影响检查

变更文件：
- [文件路径] （变更类型）

关联文件扫描结果：

✅ [文件路径] — 无需更新
⚠️ [文件路径]:L[行号] — 当前内容：[摘要] — 建议：[同步更新为 / 确认是否需要更新]

汇总：共检查 [N] 个关联文件，[N] 个需要更新，[N] 个无需变更。
```

若所有关联文件均无需更新：

```
🔍 变更影响检查

变更文件：[列表]
关联文件扫描结果：全部一致，无需同步更新 ✅
```

## 注意事项

- 只报告需要同步的位置，不自动修改，由用户确认后再改
- 对于产物变更，只检查下游（不回溯上游），因为上游是源头
- 搜索时使用变更内容的关键词，而非全文比对，避免噪音过多
