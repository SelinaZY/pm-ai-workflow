# 验收 Case 设计 Checklist

**目的**：在 PRD 第八章已按「用户场景视角」组织的前提下，设计验收 Case 时仍需做的二次推演。Case 设计不是 1:1 翻译 PRD，而是补充环境前置、攻击性视角和历史盲区。

**触发场景**：用户要求生成验收 Case 文档（如「写验收 case」「生成 acceptance case」「设计验收用例」）时，先读本文件，再开始设计。

---

## 前置假设

自 2026-05-07 起，PRD 第八章强制按「用户场景视角」组织（8.1 用户场景 + 8.2 跨场景对比 + 8.3 通用规则 + 8.4 性能）。原 checklist 中「场景推演」「对比矩阵」两维已前移至 PRD 层。Case 设计阶段的重心转向：**环境前置、攻击性视角、历史盲区回写**。

若发现接手的 PRD **仍是旧版「功能点平铺」结构**，先回到 review-prd 要求 PM 重构第八章，再开始 Case 设计。

---

## 背景：为什么保留这份 checklist

2026-04-29 首版「批量导出」验收 Case 漏掉了多个关键场景（导出全部 vs 导出已选的产物差异、全选当前页 vs 不勾选导出全部的高风险误操作、导出已选下未勾选 ≠ 跳过的语义差异）。根本原因是**当时 PRD 验收标准按功能点平铺，Case 设计只能逐条翻译**，场景推演与对比矩阵被迫后置到 Case 层。

2026-05-07 起 PRD 第八章结构重构，场景推演和对比矩阵前移到 PRD，但 Case 层仍需以下三维二次推演，避免另一类盲区。

---

## 三维强制检查

### 1. 攻击性视角（逆向思维，找数据丢失/不可逆/无兜底场景）

PRD 的用户场景主要描述「正常路径」，Case 层要主动想用户怎么「踩坑」：

- [ ] 用户最容易**误解**什么？（例如「全选 = 全部」「合并文件 = ZIP 解压后合并」）
- [ ] 用户**最痛**的失败是什么？数据丢失类 > 文案不一致 > 样式偏差
- [ ] 哪些场景**没有 UI 兜底**，只能从下载/操作结果反推？（导出无进度页、删除无二次确认）
- [ ] 不可逆操作前是否有显式提示？提示文案是否足够清晰让用户停下来？
- [ ] 边界值：0 个、1 个、阈值-1、阈值、阈值+1、最大值
- [ ] 异常输入：空内容、超长名称、特殊字符、跨权限访问
- [ ] 并发场景：多标签页同时操作、网络中断重试、用户刷新页面

### 2. 环境前置（每条 Case 自带「样本就绪」与「降级方案」）

Case 写得再细，没数据就跑不通。第一版必须前置考虑：

- [ ] 每条 Case 是否标注了所需的测试样本（项目 A/B/C... + 任务数/状态/内容特征）？
- [ ] 测试环境无法满足样本要求时，是否提供降级方案？（如：研发临时调整阈值 / 数据脚本批量克隆 / mock 慢响应）
- [ ] 阻断条件是否前置声明？（样本未就绪时 PM 验收直接退回，不浪费截图回归时间）
- [ ] 跨页/分页/大数据量类 Case，是否标注「需总数 > 单页容量」的前置条件？
- [ ] 若 PRD 场景 E（异步降级）等依赖阈值触发，是否已协调研发临时下调阈值？

### 3. 反馈回写（历史验收报告的盲区固化为本期 Case）

- [ ] 是否查阅过同类功能 / 同类交互的历史验收报告？
- [ ] 历史报告里出现过的 ❌ 不通过、⚠️ 部分通过、「样本不足」类问题，是否已转化为本期 Case 的固化检查项？
- [ ] 历史报告里 PM 复核覆盖（从 ❌ 改为 ⚠️）的「等效实现」案例，是否在本期 Case 的预期结果中明确「以下两种实现均可接受」？
- [ ] 是否有跨需求复用的共性盲区？（如所有涉及批量操作的需求都要验「全选当前页 vs 全选全部」）

---

## 输出前的最终自检

生成 Case 文档前，回头确认：

1. [ ] 每条 PRD AC 是否至少展开为 ≥2 条 Case（正向 + 边界/异常）？
2. [ ] 是否有「攻击性 Case」专门覆盖高风险误操作？
3. [ ] 是否有「样本降级方案」让验收过程不被环境卡死？
4. [ ] 自相矛盾的 Case 是否已删除？（如：前置 A 「勾选框禁用」+ 操作 B「勾选异常任务」）
5. [ ] Case 编号是否与 PRD AC 编号（`AC-A1` / `AC-X1` / `AC-G1` / `PF-01`）明确关联，可追溯？

---

## 维护说明

本 checklist 是活文档。每次验收完成后，如果 PM 验收阶段发现 Case 漏测的新场景：
- 若属于**场景/对比类盲区** → 回写到 PRD 第八章（补场景或补 8.2 跨场景对比），并提 write-prd/review-prd 的改进项
- 若属于**环境/攻击性/历史回写类盲区** → 补充本 checklist 对应维度
