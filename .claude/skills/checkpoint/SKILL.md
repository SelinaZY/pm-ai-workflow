# Checkpoint — 保存当前进度快照

用于在长会话中途保存进度，防止 API 中断导致上下文丢失。

## 执行步骤

1. **确定功能名称**：从当前工作目录或对话上下文中识别正在处理的功能名（如 `batch-export`、`perf-diagnosis`）。若无法识别，询问用户。

2. **盘点当前进度**：列出以下信息：
   - 已完成的阶段（调研 / PRD / 评审 / IxD / 原型）
   - 各阶段产出文件路径
   - 当前所处阶段及进展
   - 下一步待完成内容

3. **写入 checkpoint 文件**：
   - 路径：`workspace/checkpoints/{YYYY-MM-DD}-{feature}.md`
   - 格式见下方模板

4. **输出恢复提示**：在回复末尾给出一条用户可直接粘贴的「继续」提示语。

## Checkpoint 文件模板

```markdown
# Checkpoint：{功能名}
保存时间：{YYYY-MM-DD HH:MM}

## 已完成
- [ ] 需求调研 → {文件路径或"未开始"}
- [ ] PRD 编写 → {文件路径或"未开始"}
- [ ] 需求评审 → {文件路径或"未开始"}
- [ ] 交互设计 → {文件路径或"未开始"}
- [ ] 原型设计 → {文件路径或"未开始"}

## 当前阶段
{阶段名}：{当前进展描述，1-2句}

## 下一步
{具体待完成内容，分点列出}

## 恢复提示
> 继续 {功能名} 的 {阶段名}。已完成内容见 `workspace/checkpoints/{文件名}.md`。接下来需要：{下一步第一项}。
```

## 注意

- 若 `workspace/checkpoints/` 目录不存在，直接写入（Write 工具会自动创建）
- 同一功能多次 checkpoint 时，用日期区分，不覆盖旧文件
- 执行完毕后告知用户文件路径及恢复提示
