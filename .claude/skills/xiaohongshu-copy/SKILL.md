# 小红书文案生成 — 黛玉明察

基于产品截图信息，生成黛玉明察小红书推广文案。

## 触发条件

用户说「写小红书文案」「小红书推广」「生成小红书文案」或提供合同风险条款要求生成推广内容时触发。

## 输入

用户需提供：

1. **风险条款描述**：产品识别出的具体合同风险条款内容
2. **合同类型**：租房合同 / 劳动合同 / 宠物买卖合同 / 其他
3. **配图说明**：截图画面内容（用于文案与配图呼应）
4. **期数**（可选）：属于系列第几篇，用于系列联动

## 执行步骤

### Step 1：读取存档

读取 `outputs/daiyumingcha/marketing/小红书文案存档.md`，了解已发布的往期内容，确保：
- 新文案与往期不撞结构
- 如有系列编号则延续编号
- 了解已建立的风格基调

### Step 2：生成文案

为每个风险条款生成 **3 个版本**的文案，风格分别为：

| 版本 | 风格 | 特点 |
|------|------|------|
| A | 算账 / 系列接力型 | 用具体金额制造冲击感，适合系列联动 |
| B | 故事 / 对比反差型 | 代入感强，适合独立传播 |
| C | 短平快型 | 极简，适合快速传播 |

**文案规则：**
- 开头用 📌 + 标题（含系列编号）
- 必须包含**具体金额换算**（让用户感受到痛感）
- 产品名「黛玉明察」自然植入，不硬广
- 结尾引导行动（扫一扫 / 存起来 / 转给朋友）
- 语气：闺蜜提醒式，口语化，避免法律术语
- 目标读者：法律小白、租房/签合同的年轻人

### Step 3：用户选定后存档

用户选择版本后，将定稿追加到 `outputs/daiyumingcha/marketing/小红书文案存档.md`，格式与已有条目一致。

## 输出路径

`outputs/daiyumingcha/marketing/小红书文案存档.md`

## AI 行为规则

1. 生成前必须先读取存档，了解往期内容
2. 缺少信息时提问，不假设合同细节
3. 用户选定版本后主动存档，不需要用户额外指示
4. 如用户提供了新的合同类型场景，主动提议更新 `products/daiyumingcha.md` 的用户场景列表
